Como Funciona

El flujo completo de error a mejora del producto

1

El usuario encuentra un error en UE5

Mientras usa ClaudusBridge, ve un error en el Output Log de Unreal Engine.

2

Usa /report-bug en Discord

Abre el comando en el servidor de Galidar Studio. Un modal le pide el log, version de UE, y pasos para reproducir.

3

Bot auto-detecta el modulo

Parsea el error log y detecta cual de los 40+ modulos de ClaudusBridge esta involucrado (CBMeshManager, CBNiagaraManager, etc.)

4

Ticket creado + team notificado

Se genera un ticket (CB-001), se confirma al usuario y se postea un resumen al canal del equipo de desarrollo.

5

Dashboard: revisar y crear mejora

En el dashboard local ves todos los bugs, filtras por modulo/severidad, y creas Improvement Tasks vinculadas al codigo fuente.

6

Fix aplicado → usuario notificado

Cuando marcas el bug como "fixed" en el dashboard, el bot envia un DM al usuario que reporto el error.

A

Discord Bot - Lo que ve el usuario

Comando: /report-bug

El usuario escribe /report-bug en cualquier canal y se abre este modal:

ClaudusBridge Bug Report
Material node connections crash when using CBMaterialManager
LogClaudusBridge: Error: CBMaterialManager::CreateNode - Failed to create node 'TextureSample'
Assertion failed: NodePtr != nullptr [File: CBMaterialManager.cpp] [Line: 347]
5.7
0.2.0
1. Open Material Editor
2. Use create_material_node tool
3. Set type to TextureSample
4. Crash occurs

Respuesta del Bot

Despues de enviar, el bot responde con un embed de confirmacion:

Bug Report Created: CB-001
Material node connections crash when using CBMaterialManager
Critical
5.7
0.2.0
CBMaterialManager
Materials
Open
Track with /bug-status CB-001 | Galidar Studio
⚡
Auto-deteccion inteligente: El bot parsea palabras clave como "crash", "fatal", "assert" para severidad Critical, y busca nombres de modulos como CBMaterialManager, CBNiagaraManager, etc. en el texto del log.

Comando: /bug-status CB-001

El usuario puede consultar el estado de su reporte en cualquier momento. Muestra el estado actual, severidad, modulo detectado, y si hay mejoras vinculadas.

C

Auto-Deteccion de Modulos

El parser analiza el texto del error log y detecta automaticamente cual de los 40+ modulos de ClaudusBridge esta involucrado:

CBActorManager → Actors
CBNodeManager → BP Nodes
CBMaterialManager → Materials
CBNiagaraManager → Niagara
CBMeshManager → Mesh
CBWidgetManager → Widgets
CBSequencerManager → Sequencer
CBPhysicsManager → Physics
CBMetaSoundManager → MetaSound
CBLandscapeManager → Landscape
CBAnimationManager → Animation
CBEditorManager → Editor

+ 28 modulos mas. La severidad tambien se detecta: "crash/fatal/assert" = Critical, "error/failed" = High, "warning" = Low

Guia de Instalacion

① Clonar el repositorio

git clone https://github.com/spirit122/claudusbridge-bug-tracker.git
cd claudusbridge-bug-tracker

② Crear Discord Application

  1. Ve a discord.com/developers/applications
  2. Click "New Application" → nombre: "ClaudusBridge Bug Tracker"
  3. En el menu izquierdo, ve a "Bot"
  4. Click "Reset Token" y copia el token (guardalo seguro)
  5. Activa MESSAGE CONTENT INTENT en la seccion de Privileged Gateway Intents

③ Invitar el bot al servidor

En la seccion OAuth2 → URL Generator:

  1. Scopes: marca bot y applications.commands
  2. Bot Permissions: Send Messages, Embed Links, Use Slash Commands
  3. Copia la URL generada y abrela en el navegador
  4. Selecciona el servidor de Galidar Studio

④ Configurar variables de entorno

cp bot/.env.example bot/.env

Edita bot/.env con tus valores:

# Token del bot (del paso 2)
DISCORD_TOKEN=tu_token_aqui

# Client ID (en General Information de tu app)
CLIENT_ID=tu_client_id

# Server ID (click derecho en tu servidor → Copy Server ID)
GUILD_ID=id_del_servidor_galidar

# Canal donde el bot postea reportes al team
NOTIFICATION_CHANNEL_ID=id_del_canal_bug_reports
💡
Para obtener IDs de Discord, activa el Developer Mode en Settings → Advanced → Developer Mode. Luego click derecho en servidor/canal → Copy ID.

⑤ Instalar dependencias

cd bot && npm install
cd ../dashboard && npm install

⑥ Registrar slash commands

cd ../bot
node deploy-commands.js

Deberias ver: Successfully registered 2 commands.

⑦ Iniciar el bot

node index.js

Deberias ver: ClaudusBridge Bug Tracker bot ready!

⑧ Iniciar el dashboard

En otra terminal:

cd dashboard
node server.js

Abre http://localhost:3000 en tu navegador.

✅ Probar!

  1. En Discord escribe /report-bug
  2. Llena el modal con un error log de prueba
  3. Verifica que aparece la confirmacion con ticket ID
  4. Revisa el dashboard - el bug debe aparecer
  5. Cambia el estado, crea un Improvement Task
  6. Revisa Analytics para ver las estadisticas
